🔵 FASE 1 — Cambios de datos y reglas básicas (BACKEND, sin UI)
1️⃣ Añadir DNI/NIE a las reservas

Por qué va primero

Afecta a:

nlu

reservas.xlsx

flujos de cancelar/consultar

Es dato estructural, mejor hacerlo antes de orquestador

Qué implica:

Nueva columna en reservas.xlsx

Extracción de DNI/NIE por regex

Validación básica de formato

Slot obligatorio para reservar

2️⃣ Cancelar solo si eres el propietario (ID + email/DNI)

Muy importante a nivel de seguridad

Ahora mismo:

❌ Cualquiera puede cancelar con un ID

Objetivo:

Cancelar solo si:

ID + email o

ID + DNI/NIE

Esto debe ir antes del orquestador, porque:

Es lógica de dominio

Debe seguir siendo determinista

3️⃣ Capa de privacidad (no mostrar datos personales)

TFM point fuerte

Qué implica:

Nunca mostrar:

email

teléfono

DNI

En respuestas de consulta:

“Reserva a nombre de Juan G.”

“Contacto verificado”

Esto es regla transversal, mejor cerrarla pronto.

4️⃣ Fechas sin año → inferir año correcto

Ejemplo:

Hoy: diciembre 2025

Usuario: “09/02”

Interpretar como 09/02/2026, no 2025

Esto va aquí porque:

Afecta al NLU

Afecta a reservas y meteo

Debe estar resuelto antes del LLM

🟣 FASE 2 — Meteo y reglas de negocio
5️⃣ Meteo: añadir windspeed

Sencillo, aislado y seguro:

Solo toca:

weather_client.py

formato del resumen

Buen “warm-up” antes del orquestador.

6️⃣ Meteo: indicar dónde es el tiempo

Ahora:

“Pronóstico para esas fechas…”

Mejor:

“Pronóstico para Cobisa (Toledo)”

O “para Apartamento Mercedes (Cobisa)”

Es mejora semántica, no rompe nada.

7️⃣ Reglas de negocio: descuentos dinámicos

Este punto es oro para el TFM.

Ejemplo:

Faltan 10 días → −5 %

Faltan 5 días → −10 %

Faltan 2 días → −20 %

Por qué va aquí:

Afecta a:

precio mostrado

resumen de reserva

Conviene hacerlo antes del orquestador, para que el LLM no invente precios

🟢 FASE 3 — ORQUESTADOR (NÚCLEO DE IA)
8️⃣ Orquestador LLM (GPT-4o mini)

Este es el gran salto conceptual.

Rol del orquestador:

Decidir intención cuando el regex duda

Reescribir la consulta en formato estructurado

Elegir:

info_bot

sql_bot

meteo_bot

o combinación

Importante:

❌ NO sustituye reglas

✅ Las coordina

Este va después de:

DNI

fechas

seguridad

reglas de precio

Así el LLM no “rompe” nada.

9️⃣ Idioma automático (responde en el idioma del usuario)

Esto encaja perfectamente con el orquestador:

Detectar idioma

Pasarlo como instrucción al LLM

Mantener backend en español internamente

Muy buen detalle académico.
